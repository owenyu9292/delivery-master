# 배송마스터 시즌2 안전 계획

## 기본 원칙

- `main`은 매일 사용하는 실사용 앱으로 보호한다.
- `stable-2026-05-13`은 정상 기준점으로 보존한다.
- GitHub `main`에 바로 올리지 않는다.
- 수정은 반드시 별도 브랜치에서 먼저 한다.
- 구조 변경보다 localStorage 데이터 보호를 우선한다.
- 보고서, 작업 문서, 설명은 한글로 작성한다.

## 1단계: 안전장치 먼저

- 오늘 초기화 전에 자동 백업을 만든다.
- 가져오기 덮어쓰기 전에 자동 백업을 만든다.
- 현재 화면 형태와 전역 함수 이름은 최대한 유지한다.
- 앱 코드가 바뀌면 서비스워커 캐시 버전을 올린다.

## 2단계: 내보내기와 복구 강화

- 내보내기에 진행 중 상태, 로그, 리포트, 요약, 설정을 함께 담는다.
- 앱 시작 오류가 나도 최소 백업/내보내기 화면은 뜨게 한다.
- 자동 안전 백업 목록을 확인할 수 있게 한다.
- 복구 기능은 백업 형식이 안정된 뒤 추가한다.
- 현장에서 코과장에게 바로 연결하지 못하는 상황을 대비해 필드 메모와 진단 내보내기 기능을 만든다.

## 3단계: 테스트 앱 분리

- 테스트용 앱 경로나 저장소를 따로 만든다.
- 테스트 앱은 테스트 전용 localStorage 키를 쓴다.
- 테스트 앱은 별도 서비스워커 캐시 이름을 쓴다.
- 테스트 앱이 실사용 데이터 키를 건드리지 않는지 확인한다.

## 4단계: 의존관계 지도 만들기

- 전역 상태를 정리한다: `S`, `logs`, `zoneCfg`, `helperType`, `pendingQty`.
- HTML `onclick`에서 직접 부르는 전역 함수를 정리한다.
- 파일 간 함수 호출 관계를 정리한 뒤 코드를 옮긴다.
- 옮기는 동안 기존 함수명은 호환용으로 남긴다.

## 5단계: 기능 기준으로 다시 나누기

- 상태와 저장.
- 업무 진행 흐름.
- 리포트 생성과 리포트 기록.
- 백업, 가져오기, 내보내기, 복구.
- 화면 전환, 모달, 토스트.
- 설정: 구역, Gmail, 휴무.
